# UI 窗口框架示例

这个示例展示如何用 DSPCore UI 框架创建一个最小自定义窗口壳，并用声明式网格放置页头、内容卡片、按钮和勾选框。

## 适用场景

- 模组需要自己的设置窗口、信息窗口或工具窗口。
- 需要复用 DSPCore 的窗口生命周期、基础控件和网格布局。
- 不需要把页面注册到游戏原版物品、配方或制造器分页中；那些入口属于 `Tabs` 功能块。

## 关键点

- `UiWindowManager.CreateWindow<T>()` 只创建并登记窗口，不自动决定打开时机。
- 窗口内容通常在 `_OnCreate` 或模组自己的初始化流程中创建。
- `GridDsl.BuildLayout(...)` 只负责把布局节点投射成 Unity `RectTransform`。
- `GridDsl.FormCard(...)`、`ListCard(...)`、`DetailCard(...)` 和 `StatusFooter(...)` 可以减少常见工具页的重复布局代码。
- 具体按钮行为、配置保存、解锁逻辑和业务页面仍由模组自己处理。

## 常见误用

- 不要把 `UiTabbedWindow` 和游戏原版 `Tabs` 功能块混用。前者是自定义窗口内部页签，后者是物品/配方/制造器分页。
- 不要在 UI 功能块里注册具体业务页面；UI 只提供框架和控件。
- 不要假设 `CreateWindow` 会打开窗口；需要调用窗口实例的 `Open()`。
